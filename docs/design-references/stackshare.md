# StackShare Reference Notes

Role: technology comparison, stack metadata, and tool-discovery inspiration.

## Useful Patterns

- Technology names, categories, and metadata need to remain readable at a glance.
- Comparison views benefit from compact badges and consistent category labels.
- Discovery flows should make saved/dismissed state obvious.
- Tooling context matters: languages, topics, GitHub signals, and evidence sources should stay near the item.

## RepoRadar Translation

- Keep repository language/topics and evidence near the repo title.
- Make saved, read, ignored, candidate, and dismissed states visible without extra navigation.
- Use badges for discovery profiles, source type, score, and risk.
- Preserve long names and descriptions with truncation/clamping rather than layout shift.

## Boundaries

- Do not recreate StackShare's public-directory or social proof model.
- RepoRadar remains local-first and decision-oriented, not a public catalogue.
