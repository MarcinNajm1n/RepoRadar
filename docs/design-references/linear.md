# Linear Reference Notes

Role: workflow density and issue navigation inspiration.

## Useful Patterns

- Quiet shell navigation with clear current location.
- Compact rows where status, owner, priority, and title can be scanned together.
- Strong keyboard and focus affordances without visual noise.
- Muted metadata that supports the title instead of competing with it.

## RepoRadar Translation

- Treat repository and task cards like work items in a signal ledger.
- Keep labels short and action-oriented.
- Use low-contrast borders and small radius for technical precision.
- Make active navigation obvious with subtle surface and accent changes.

## Boundaries

- Do not copy Linear's brand, gradients, icon treatment, or exact color palette.
- RepoRadar should remain more data/research oriented than project-management oriented.
