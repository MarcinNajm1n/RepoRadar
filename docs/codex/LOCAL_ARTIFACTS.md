# Local Artifacts

RepoRadar keeps project instructions and reusable workflow files in Git, but
runtime outputs and user-specific editor/tool files must stay local.

## Tracked Project Files

- `.codex/` stores project agent and hook configuration that is part of the
  RepoRadar Codex workflow.
- `.agents/` stores project-local skills used by the repository workflow.
- `docs/codex/` stores workflow documentation for future Codex runs.
- `.githooks/pre-commit` runs the staged sensitive-file check and is safe to
  version because it does not contain user-specific absolute paths.

## Local-Only Files

- `graphify-out/` is generated Graphify output. Regenerate it locally; do not
  commit it unless a future task explicitly changes that policy.
- `.obsidian/` is personal editor workspace state.
- `.impeccable/` contains local design-tool state if generated.
- `.githooks/post-commit` and `.githooks/post-checkout` may be generated by
  Graphify and can contain local Python paths. Treat them as machine-specific
  hook installs, not shared project files.

## Handling Rules

- Stage only explicit files for commits.
- Keep generated graph, editor, and machine-specific hook files out of Git.
- If a local artifact becomes intentionally shared, remove it from `.gitignore`
  in the same commit that adds the documented shared workflow.
